Hardware
Pico + CC1101 + 433 MHz remote
Goal
Capture one fixed code and replay it

Workshop Flow

Set Up

Open Thonny, select the Pico MicroPython interpreter, and verify the board is connected.

2

Capture

Run the sniffer, press the remote once, and copy the printed addr and btn.

3

Replay

Insert those values into the transmit step and verify the target device responds.

Quick Wiring Table

Important

Use 3.3V only. The CC1101 is not 5V tolerant.

Thonny Setup

  • Install/open thonny.org
  • Select MicroPython (Raspberry Pi Pico)
  • Open this repo and run blink.py first

Code Snippet 1 - Blink Test

# blink.py
from machine import Pin
import time

led = Pin(25, Pin.OUT)
print("Hello from Pico!")

while True:
    led.toggle()
    print("Blink!")
    time.sleep(0.5)

Expected result: LED blinks and Thonny prints Blink!.

Code Snippet 3 - Replay

# Step 2: Replay captured code
DEVICE_ADDRESS = 0xBCC25  # replace with your addr
BUTTON_1 = 0x2            # replace with your btn
transmit(cc1101, DEVICE_ADDRESS, BUTTON_1)

Use the values printed by the sniffer. Then run the script again.

Code Snippet 2 - Capture

# Step 1: Sniff the signal
result = sniff(cc1101)
if result:
    addr, btn, code = result
    print(f"[RESULT] Captured code: 0x{code:06X}")
    print(f"         Use: transmit(cc1101, 0x{addr:05X}, 0x{btn:X})")

Press the remote once during the capture window.

Capture Notes

  • Press the remote once, not repeatedly
  • Write down both addr and btn
  • If it fails, bring the remote closer and try again
  • One clean capture is enough for replay
